14/*
15 * This class is to support application table reconciliation
16 * For any application table which has entries with key -> vector<f1/v2, f2/v2..>
17 * It is expect that the application owner keeps the order of f/v pairs
18 * The application usually takes this class as composition, i.e. includes an instance of
19 * this class in their classes.
20 * A high level flow to use this class:
21 * 1. Include this class in the application class: appClass.
22 *
23 * 2. Construct appClass along with this class with:
24 * docker name, application name, timer value etc
25 *
26 * 3. Define Select s;
27 * Check if warmstart enabled, if so,read application table to cache and start timer:
28 * if (appClass.getRestartAssist()->isWarmStartInProgress())
29 * {
30 * appClass.getRestartAssist()->readTableToMap()
31 * appClass.getRestartAssist()->startReconcileTimer(s);
32 * }
33 *
34 * 4. Before the reconcile timer is expired, insert all requests into cache:
35 * if (m_AppRestartAssist.isWarmStartInProgress())
36 * {
37 * m_AppRestartAssist.insertToMap(key, fvVector, delete_key);
38 * }
39 *
40 * 5. In the select loop, check if the reconcile timer is expired, if so,
41 * stop timer and call the reconcilation function:
42 * Selectable *temps;
43 * s.select(&temps);
44 * if (appClass.getRestartAssist()->isWarmStartInProgress())
45 * {
46 * if (appClass.getRestartAssist()->checkReconcileTimer(temps))
47 * {
48 * appClass.getRestartAssist()->stopReconcileTimer(s);
49 * appClass.getRestartAssist()->reconcile();
50 * }
51 * }
52 */
