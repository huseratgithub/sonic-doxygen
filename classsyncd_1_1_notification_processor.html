◆ check_fdb_event_notification_data()

bool NotificationProcessor::check_fdb_event_notification_data ( _In_ const sai_fdb_event_notification_data_t & data)
private

@Brief Check FDB event notification data.

Every OID field in notification data as well as all OID attributes are checked if given OID (returned from ASIC) is already present in the syncd local database. All bridge ports, vlans should be already discovered by syncd discovery logic. If vendor SAI will return unknown/invalid OID, this function will return false.

Parameters
dataFDB event notification data
Returns
False if any of OID values is not present in local DB, otherwise true.
