Watchdog scope constructor.

Parameters
timerWatchdog- Timer watchdog to work on.
eventName- Name of the event.
data- Event data. NOTE: event data is passed here as a pointer here to avoid expensive data copy, but then we must be sure that that data object was not destroyed before watchdog scope destructor will be called. This will make sure that if event was too long, then it data will be logged into syslog from object that is still alive and not destroyed.
